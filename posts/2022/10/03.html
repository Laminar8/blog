New SvelteKit
One day in May, this blog has been started from cloning new sveltekit. While creating blog layouts and posting some, sveltekit has new changes. Directories and files in your app are defined different in comparison with old sveltekit although sveltekit uses same filesystem-based router systems.
I don't know exactly the sveltekit version, but at least there was no rule or guideline to add plus sign to my svelte files in my first app. However new sveltekit defines new a page of your app with a
+page.svelte
component file. Similary
+page.js
and
+page.server.js
files can be defend to load some data.
With git, I would like to migrate blog data from old sveltekit app to new one. You can think this migration plan is not suited because this blog is only managed by me. However let's guess the blog is managed from a group members.

Git branch
I'm currently using a
dev
branch. For upgrading my sveltekit, I create a new branch,
feature
. If my app on the
feature
branch works properly, the branch would be merged to the
dev
branch.
Bash
git branch feature
Following above bash command, you can create a new branch.
Bash
git checkout feature
You can switch to your new branch,
feature
.
Bash
npm create svelte@latest my-app
cd my-app
npm install
To clone a new sveltekit app, follow the official commands.
Bash
npm install -D sass gh-pages ...
You also have to add your dev dependencies on your
package.json
file.

feature

Advanced routing
While applying new svelteKit to my project, I couldn't understand
Advanced routing 1
. My blog is serving all of my posting correctly, however I'm not sure I can create another blog.
1
SvelteKit needs to know which route is being requested. The app are routed using high priority rules, such as rest parameters, matching, sorting, and so on.
Route hierarchy and layout hierarchy are very complicated. Valid directories from old app couldn't be compatible with new app maybe. Be careful this routing system.
